# ce-remotion

A Remotion video explaining **Context Engineering (CE) fundamentals** with Claude Code.

Built using the very workflow it demonstrates — every scene and component is developed via Claude Code with a full CE setup (CLAUDE.md, decisions.md, task.md, PRPs, auto-commit on every feature).

---

## What's In The Video

| Scene | Topic |
|---|---|
| 1 | The problem with vague AI prompts |

## Project Structure

```
ce-remotion/
├── CLAUDE.md                    # Claude Code rules and conventions
├── docs/
│   ├── decisions.md             # Confirmed design and architecture decisions
│   ├── task.md                  # Technical gotchas and implementation patterns
│   ├── planning.md              # Scene storyboard and build order
│   └── testing.md               # Review and QA checklist
├── init/                        # Feature specs (authored in Claude.ai)
├── prp/                         # Product Requirements Prompts (generated by Claude Code)
├── .claude/commands/            # /generate-prp and /execute-prp commands
├── src/
│   ├── Root.tsx                 # Composition registry
│   ├── Master.tsx               # Scene sequencing
│   ├── scenes/                  # One file per scene
│   └── components/              # Reusable components
└── public/                      # Static assets
```

## Development

```bash
# Install dependencies
npm install

# Start Remotion Studio (localhost:3000)
npm run dev

# Render final video
npx remotion render Master out/ce-remotion.mp4
```

Requires Node 18+ (use `nvm`).

---

## CE Workflow Used To Build This

1. **Claude.ai + Human** write an `init/` file describing the feature
2. **Claude Code** runs `/generate-prp` to produce a scoped implementation plan
3. **Human** reviews the PRP in Claude.ai before approving
4. **Claude Code** runs `/execute-prp` to implement the feature
5. **Claude Code** auto-commits and pushes on completion
